# 방수명가 자체 CRM 요구사항 정의서

**작성일:** 2026-04-04
**목적:** 세일즈맵 UX를 베이스로, 방수 시공업 특화 자체 CRM 구축
**데이터베이스:** Supabase (기존 광고 대시보드와 동일 프로젝트)

---

## 1. 개요

### 1.1 현재 상태
- 노션 DB로 CRM 운영 중 → 채널 분류 오류, 자동화 불가, 분석 한계
- 광고 대시보드(ad-dash)가 별도로 존재하며, CRM 데이터를 valid_snapshot 테이블로 수동 동기화
- 대시보드와 CRM이 같은 Supabase를 쓰면 동기화 자체가 불필요해짐

### 1.2 핵심 목표
- 직원 1건 등록 시간: **30초 이내**
- 모바일(스마트폰)에서 현장 입력 가능
- 광고 대시보드와 실시간 데이터 공유
- 제안서/적외선 측정 등 영업 도구의 효과를 데이터로 검증 가능하게

### 1.3 UX 레퍼런스
- **세일즈맵(salesmap.kr):** 파이프라인 관리 UI, 딜 상세 히스토리, 직관적 드롭다운
- **파이프드라이브(Pipedrive):** 칸반 보드 파이프라인, 모바일 UX
- 두 서비스의 파이프라인 뷰 + 모바일 입력 흐름을 참고하되, 필드와 워크플로우는 아래 방수 업종 맞춤 설계를 따름

## 2. 데이터 구조

### 2.1 핵심 오브젝트: 문의(Inquiry)

하나의 문의 = 하나의 현장. 같은 고객이 다른 현장을 의뢰하면 별도 문의로 등록.

#### 자동 기록 필드 (직원 입력 불필요)

| 필드명 | 타입 | 설명 | 자동화 방법 |
|--------|------|------|-------------|
| id | UUID | 고유 식별자 | 자동 생성 |
| created_at | timestamp | 문의 접수 시각 | 카드 생성 시 자동 |
| channel | enum | 유입 채널 | UTM 자동 태깅 또는 소스별 자동 (아래 상세) |
| utm_medium | text | 매체 구분 | URL 파라미터에서 자동 추출 |
| utm_keyword | text | 유입 키워드 | URL 파라미터에서 자동 추출 |
| first_response_at | timestamp | 직원 첫 열람 시각 | 카드 최초 오픈 시 자동 기록 |
| updated_at | timestamp | 마지막 수정 시각 | 자동 |
| stage_changed_at | timestamp | 파이프라인 단계 변경 시각 | 단계 변경 시 자동 |

#### 직원 입력 필드 — 필수 (카드 생성 시)

| 필드명 | 타입 | 입력 방식 | 설명 |
|--------|------|-----------|------|
| address | text | 텍스트 입력 | **주 식별자**. 현장 주소 (예: "래미안 아파트 301동") |
| client_name | text | 텍스트 입력 | 고객명 또는 대명사 (예: "삼성 래미안", "옥상 아저씨") |
| phone | text | 텍스트 입력 | 연락처 |
| work_type | enum | 드롭다운 | 시공 분야: 옥상 / 외벽 / 화장실 / 지하 / 베란다 / 기타 |

#### 직원 입력 필드 — 선택 (진행 중 업데이트)

| 필드명 | 타입 | 입력 방식 | 시점 |
|--------|------|-----------|------|
| estimate_amount | integer | 숫자 입력 | 견적 산출 시 |
| contract_amount | integer | 숫자 입력 | 계약 확정 시 |
| area_sqm | number | 숫자 입력 | 시공 평수 (선택) |
| memo | text | 텍스트 | 자유 메모 |

#### 체크박스 필드 (탭 한 번)

| 필드명 | 타입 | 설명 | 분석 목적 |
|--------|------|------|-----------|
| proposal_sent | boolean | 제안서 발송 완료 | 제안서 발송 건 vs 미발송 건 전환율 비교 |
| ir_inspection | boolean | 적외선 측정 실시 | 적외선 측정 건 vs 미실시 건 전환율 비교 |
| case_documented | boolean | 시공 사례 정리 완료 | 포트폴리오/블로그 콘텐츠 관리 |

### 2.2 파이프라인 단계 (pipeline_stage)

현행 노션 CRM의 단계를 그대로 유지. 이 단계는 현장에서 최적화된 것이므로 축소하지 않음.

```
문의접수 → 연락대기 → 견적서전송 → 한달이상걸림 → 계약 중 → 착수금 대기 → 착수금완료 → 공사중 → 잔금대기 → 잔금완료 → 정산 대기 → 정산 완료 → 보수 완료
```

파이프라인 뷰는 칸반 보드 형태로 표시 (세일즈맵/파이프드라이브 스타일).
각 단계에 머무는 시간을 자동 계산하여, 정체된 딜을 시각적으로 강조.

### 2.3 계약 상태 (contract_status)

| 값 | 설명 |
|-----|------|
| 진행중 | 아직 결과 미정 (기본값) |
| Won / 계약성공 | 계약 확정 |
| Lost | 실패/거절 |

**계약 판정 로직 (대시보드 연동용):**
- Won 조건: contract_status = 'Won' 또는 '계약성공' 또는 pipeline_stage가 '계약 중' 이후 단계
- Lost 조건: contract_status = 'Lost'
- 진행중: 위 두 가지 어디에도 해당하지 않음

### 2.4 채널 (channel) enum

```
naver_powerlink    — 네이버 파워링크 (UTM 자동)
naver_powercontent — 네이버 파워컨텐츠 (UTM 자동)
naver_phone        — 네이버 전화 (스마트콜 자동 또는 수동)
soomgo             — 숨고 (수동 선택)
daum_nate          — 다음/네이트 (수동)
referral           — 소개/재시공 (수동)
etc                — 기타 (수동)
```

## 3. 채널 자동 태깅 로직

### 3.1 네이버 폼 (웹훅 연동)

네이버 광고 랜딩 URL에 UTM 파라미터 설정:
```
파워링크:    ?utm_source=naver&utm_medium=powerlink&utm_keyword={keyword}
파워컨텐츠:  ?utm_source=naver&utm_medium=powercontent&utm_keyword={keyword}
```

문의 폼 제출 시 UTM 값을 함께 전송 → CRM 카드 자동 생성 + channel 자동 설정.
`{keyword}`는 네이버 광고의 자동 치환 매크로로, 실제 검색 키워드가 들어감.

### 3.2 네이버 전화 (스마트콜 연동)

네이버 스마트콜 통화 기록(전화번호, 시간, 통화시간) → CRM 카드 자동 생성.
channel = 'naver_phone' 자동 설정.
직원은 카드를 열어서 주소 + 시공 분야만 입력.

**슈퍼앱 팀 확인 필요:** 네이버 스마트콜 API의 데이터 제공 범위 및 웹훅 지원 여부.

### 3.3 숨고

숨고 API 없음. 자동 연동 불가.
직원이 "새 문의" 버튼 클릭 시 채널 드롭다운에서 "숨고" 선택.
UX: "숨고" 선택 시 숨고 앱에서 복사할 정보(고객명, 연락처)를 빠르게 붙여넣을 수 있는 입력 흐름.

### 3.4 소개/재시공

직원이 채널 드롭다운에서 "소개/재시공" 선택.
추가 필드: referral_source (텍스트, 선택) — 누구 소개인지 (예: "래미안 301동 김씨")

## 4. 입력 흐름 (시나리오별)

### 4.1 네이버 폼 문의 (가장 빈번)
```
[자동] 웹훅 → CRM 카드 생성 (channel, keyword, 고객 입력 정보 자동)
[알림] 직원에게 푸시 알림
[직원] 카드 열기 → 주소 입력 → 시공 분야 선택 → 저장
소요시간: ~10초
```

### 4.2 네이버 전화 (스마트콜 연동 시)
```
[자동] 스마트콜 통화 종료 → CRM 카드 생성 (channel, 전화번호 자동)
[직원] 카드 열기 → 고객명 + 주소 + 시공 분야 입력 → 저장
소요시간: ~15초
```

### 4.3 숨고 문의
```
[직원] "새 문의" 버튼 → 채널 "숨고" 선택 (기본 선택 가능)
       → 고객명 + 연락처 + 주소 + 시공 분야 입력 → 저장
소요시간: ~20초
```

### 4.4 견적 후 업데이트
```
[직원] 해당 카드 → 파이프라인 "견적서전송"으로 이동
       → 견적 금액 입력 → ☐ 제안서 발송 체크 → 저장
소요시간: ~5초
```

### 4.5 방문 후 업데이트
```
[직원] 해당 카드 → ☐ 적외선 측정 실시 체크 → 메모 추가 (선택) → 저장
소요시간: ~2초
```

### 4.6 시공 완료
```
[직원] 파이프라인 단계 이동 → ☐ 시공 사례 정리 완료 체크 → 사진 첨부 (선택)
소요시간: ~10초
```

## 5. 대시보드 연동

### 5.1 데이터 공유 구조

CRM과 광고 대시보드가 같은 Supabase 프로젝트에 있으므로, 별도 동기화 불필요.
대시보드가 CRM 테이블을 직접 쿼리.

**기존 valid_snapshot 테이블 → 점진적 교체:**
- 1단계: CRM 완성 후에도 valid_snapshot은 유지. 대시보드는 valid_snapshot을 계속 사용.
- 2단계: 대시보드의 데이터 소스를 CRM 테이블로 전환 (추상화 레이어 교체).
- 3단계: valid_snapshot 폐기.

### 5.2 대시보드가 CRM에서 가져가는 데이터

| 대시보드 지표 | CRM 소스 |
|---------------|----------|
| 유효 문의 건수 | 전체 inquiry 건수 |
| 채널별 문의 | channel 필드 group by |
| 계약 건수 | contract_status = 'Won' |
| 계약 금액 | contract_amount (없으면 estimate_amount 폴백) |
| 계약 전환율 | Won / 전체 |
| 딜 상태 (확정/미확정) | contract_status별 집계 |
| 제안서 효과 분석 | proposal_sent 그룹별 전환율 비교 |
| 적외선 효과 분석 | ir_inspection 그룹별 전환율 비교 |
| 코호트 전환 추적 | created_at 월별 → 최종 contract_status 추적 |

## 6. 알림

### 6.1 직원 알림
- 새 문의 접수 시 푸시/카카오톡 알림
- 파이프라인 "견적서전송" 후 7일 경과 시 "상태 업데이트 필요" 알림
- 파이프라인 "한달이상걸림" 상태 건 주 1회 리마인드

### 6.2 경영자 알림 (대시보드 연동)
- 유효CPA가 임계치 초과 시 알림
- 일 광고비가 전일 대비 200% 초과 시 알림

## 7. 기존 데이터 마이그레이션

### 7.1 valid_snapshot (587건) → CRM 테이블
- 기존 데이터의 채널이 혼재 ('네이버 - 폼', 'naver_form' 등) → 정규화 필요
- 마이그레이션 시 채널명 매핑 테이블 적용
- contract_amount, estimate_amount 등 금액 데이터 보존
- 기존 480건의 contract_amount는 절대 수정하지 않음

### 7.2 채널 정규화 매핑

| 기존 값 | 정규화 값 |
|---------|-----------|
| 네이버 - 폼, naver_form | naver_powerlink (UTM 구분 불가 시 기본값) |
| 네이버 - 전화, naver_phone | naver_phone |
| 숨고, soomgo | soomgo |
| 기타, etc, 채널톡 | etc |

## 8. 모바일 요구사항

- 반응형 웹 또는 PWA (앱 설치 없이 브라우저로 접근)
- 모든 입력 필드의 탭 타겟: 최소 48×48px
- 드롭다운은 네이티브 select 또는 바텀 시트 (모바일에서 스크롤 가능)
- 카드 생성 → 필수 필드 입력 → 저장이 한 화면 안에서 완료
- 사진 첨부: 카메라 직접 촬영 또는 갤러리에서 선택

## 9. 우선순위

| 순위 | 기능 | 이유 |
|------|------|------|
| 1 | 문의 카드 CRUD + 파이프라인 칸반 | CRM의 핵심 |
| 2 | 채널 드롭다운 + 체크박스 (제안서/적외선) | 분석 데이터 수집의 기반 |
| 3 | 네이버 폼 웹훅 → 카드 자동 생성 | 직원 입력 부담 최소화 |
| 4 | 대시보드 연동 (valid_snapshot 대체) | 데이터 일원화 |
| 5 | 스마트콜 연동 | 전화 문의 자동화 |
| 6 | 알림 시스템 | 딜 관리 누락 방지 |
| 7 | 사진 첨부 + 시공 사례 관리 | 포트폴리오 자동 축적 |
