**Problem 1: skipLlm stale 값**

`EstimateEditor.tsx` L178에서 `skipLlm: voiceFlowRef.current.isActive`가 렌더 시점에 평가된다.
useVoice 내부에서 `skipLlmRef`로 ref 동기화하고 있어서(L61, L64) 실제로는 ref가 최신값을 추적한다.
그러나 `voiceFlowRef.current.isActive`는 이전 렌더의 값이다.

해결: `EstimateEditor.tsx`에서 `skipLlm`을 `voiceFlow.isActive`로 직접 전달하지 말고, `useVoice`의 `onSttText` 콜백 내에서 voiceFlow 활성 여부를 판단하는 현재 패턴을 강화한다.

구체적 변경 (`components/estimate/EstimateEditor.tsx`):

1. L178의 `skipLlm: voiceFlowRef.current.isActive`를 제거하고, 대신 `skipLlm`을 항상 `false`로 두되, `onSttText` 콜백에서 voiceFlow 활성 중이면 processText만 호출하고 return하는 현재 패턴(L179-185)을 유지한다. 단, `skipLlm`도 함께 true로 세팅해야 useVoice가 LLM 호출을 건너뛴다.

실제 해결책: `voiceFlow.isActive`를 state로 추적하는 별도 useState를 도입한다.

```typescript
const [flowActive, setFlowActive] = useState(false)
```

useVoiceFlow가 isActive 변경될 때마다 동기화:
```typescript
useEffect(() => {
  setFlowActive(voiceFlow.isActive)
}, [voiceFlow.isActive])
```

그리고 useVoice options에:
```typescript
skipLlm: flowActive,
```

이렇게 하면 `flowActive`가 React state이므로 re-render를 트리거하고, useVoice 내부의 `skipLlmRef`가 최신값으로 갱신된다.

**Problem 2: voiceFlowRef.current.processText stale 참조**

`voiceFlowRef.current`는 L216에서 매 렌더마다 갱신되지만, `onSttText` 콜백은 useVoice 내부의 `callbacksRef`에 저장되므로 항상 최신 voiceFlowRef를 참조한다. 이 부분은 안전하다 — 변경 불필요.

**Problem 3: useVoiceFlow 내부 stateRef/useState 동기화**

`hooks/useVoiceFlow.ts`의 `updateState` 함수(L40-43)는 stateRef와 setState를 동시에 업데이트한다. processText(L57)에서는 `stateRef.current`를 읽어 항상 최신 상태를 본다. 이 패턴은 올바르다.

단, `processText`와 `goToNextEmpty`가 useCallback 의존성에 `callbacks`만 있고 `stateRef`는 ref이므로 의존성 배열에 없다 — 이것은 의도된 것이며 올바르다.

추가 안정화: `processText` 내에서 `stateRef.current.step === 'idle' || stateRef.current.step === 'done'`일 때 early return하는 가드를 추가한다. 이렇게 하면 플로우가 끝난 후 늦게 도착하는 STT 결과가 무시된다.

```typescript
// processText 함수 시작 부분에 추가
const current = stateRef.current
if (current.step === 'idle' || current.step === 'done') return
```
