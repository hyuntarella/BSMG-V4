Notion 환경변수 (이미 .env.local에 설정됨):
- NOTION_CRM_TOKEN: Notion integration token
- NOTION_CRM_DB_ID: CRM 데이터베이스 ID
- NOTION_CRM_DATA_SOURCE_ID: (필요 시 사용)

Notion CRM 필드 매핑:
- Title: 주소 (primary key)
- Rich Text: 고객명, 시공평수
- Phone: 전화번호
- Select: 담당자 (이창엽/박민우/미배정), 단계, 파이프라인, 계약상태, 문의채널
- Multi-select: 시공분야
- Number: 견적금액, 계약금액, 착수금, 잔금
- Date: 문의일자, 견적방문일자, 잔금완료, 견적서발송일, 견적서열람일
- URL: 구글드라이브URL, 견적서웹URL
- Email: 고객이메일

5 stages with pipelines:
- 0.문의: 정보입력완료, 견적일정확정, 견적방문완료, 시간좀걸림, 보류
- 1.영업: 먼저연락X, 견적서전송, 성공확률↓, 가견적전송, 성공확률↑
- 1-1.장기: 대기1~3, 내년공사희망, 재연락금지
- 2.시공: 계약중, 착수금완료, 공사중, 잔금대기, 잔금완료
- 3.하자: 하자접수, 보수일정조율중, 보수중, 보수완료

기존 패턴 참조:
- API route: app/api/stt/route.ts, app/api/llm/route.ts 패턴
- Server client: lib/supabase/server.ts 패턴
- 미들웨어: middleware.ts (auth 검증)

   - CrmRecordCreate: Partial<CrmRecord> 에서 address 필수
   - CrmRecordUpdate: Partial<CrmRecord>
   - CrmComment: id, content, createdTime, createdBy
   - NotionPage: Notion API raw page 타입 (properties 매핑용)
   - STAGE_MAP: Record<string, string[]> — 단계별 파이프라인 목록 상수
     ```
     '0.문의': ['정보입력완료', '견적일정확정', '견적방문완료', '시간좀걸림', '보류'],
     '1.영업': ['먼저연락X', '견적서전송', '성공확률↓', '가견적전송', '성공확률↑'],
     '1-1.장기': ['대기1', '대기2', '대기3', '내년공사희망', '재연락금지'],
     '2.시공': ['계약중', '착수금완료', '공사중', '잔금대기', '잔금완료'],
     '3.하자': ['하자접수', '보수일정조율중', '보수중', '보수완료']
     ```
   - PIPELINE_TO_STAGE: Record<string, string> — 파이프라인 -> 단계 역매핑 (모든 파이프라인에 대해)

2. lib/notion/client.ts 생성:
   - Notion REST API를 fetch로 직접 호출 (SDK 설치 불필요).
   - notionFetch(endpoint, method, body?) 헬퍼: `https://api.notion.com/v1${endpoint}`, headers에 Authorization: Bearer NOTION_CRM_TOKEN, Notion-Version: 2022-06-28, Content-Type: application/json.
   - export { notionFetch }

3. lib/notion/crm.ts 생성:
   - parseNotionPage(page: NotionPage): CrmRecord — Notion property 타입별 파싱:
     - title → page.properties.주소.title[0]?.plain_text
     - rich_text → .rich_text[0]?.plain_text
     - phone_number → .phone_number
     - select → .select?.name
     - multi_select → .multi_select.map(s => s.name)
     - number → .number
     - date → .date?.start
     - url → .url
     - email → .email
     - created_time, last_edited_time → page 레벨
   - getAllRecords(): Promise<CrmRecord[]> — POST /databases/{NOTION_CRM_DB_ID}/query, filter archived=false, sorts: last_edited_time desc. 페이지네이션 처리 (has_more + start_cursor). 각 page를 parseNotionPage로 변환.
   - getPageById(id: string): Promise<CrmRecord> — GET /pages/{id}, parseNotionPage.
   - updateRecord(id: string, data: CrmRecordUpdate): Promise<void> — PATCH /pages/{id}, buildNotionProperties(data) 호출.
   - createRecord(data: CrmRecordCreate): Promise<CrmRecord> — POST /pages, parent: { database_id }, properties: buildNotionProperties(data). 반환값 parseNotionPage.
   - archiveRecord(id: string): Promise<void> — PATCH /pages/{id}, { archived: true }.
   - getPageComments(id: string): Promise<CrmComment[]> — GET /comments?block_id={id}, 파싱.
   - addComment(pageId: string, content: string): Promise<CrmComment> — POST /comments, { parent: { page_id }, rich_text: [{ text: { content } }] }.
   - buildNotionProperties(data: CrmRecordUpdate): object — CrmRecord 필드를 Notion property 포맷으로 역변환. 각 필드가 undefined이면 skip.
