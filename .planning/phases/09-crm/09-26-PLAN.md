1. KanbanCard.tsx 드래그 지원 추가:
   - draggable={true}
   - onDragStart: dataTransfer.setData('text/plain', record.id). 드래그 시 opacity-50.
   - onDragEnd: opacity 복원.

2. KanbanBoard.tsx 드롭 영역:
   - 각 파이프라인 컬럼에 onDragOver(e.preventDefault), onDrop 핸들러.
   - onDrop: dataTransfer에서 recordId 추출. 해당 컬럼의 pipeline명 획득.
   - 같은 파이프라인이면 무시.
   - 다른 파이프라인이면:
     a. 이전 상태 저장 (prevPipeline, prevStage).
     b. 낙관적 UI 업데이트: updateRecordLocal(id, { pipeline: newPipeline, stage: newStage }).
        (PIPELINE_TO_STAGE로 새 stage 결정. 같은 stage 내 이동이면 stage 유지.)
     c. PATCH /api/crm/[id] 호출 (pipeline + stage 변경).
     d. 5초 Undo 토스트 표시.
   - 드롭 영역 시각 피드백: onDragOver 시 bg-blue-50 border-blue-200 border-dashed.

3. Undo 토스트:
   - KanbanBoard 내부 상태: undoInfo: { recordId, prevPipeline, prevStage, timer } | null.
   - 토스트 UI: fixed bottom-20 left-1/2 -translate-x-1/2, bg-gray-800 text-white rounded-lg px-4 py-3 shadow-xl.
   - 텍스트: "{주소} → {newPipeline}로 이동" + "되돌리기" 버튼.
   - 되돌리기 클릭: PATCH /api/crm/[id] (prevPipeline, prevStage) + updateRecordLocal + 토스트 닫기.
   - 5초 후 자동 닫기 (clearTimeout on unmount).
   - 새 이동 발생 시 이전 Undo 타이머 클리어 + 새 토스트.

4. hooks/useCrm.ts에 updateRecord 함수 추가:
   - updateRecord(id, partial): PATCH /api/crm/[id] + updateRecordLocal. 에러 시 rollback.

5. 상세 모달(DetailModal)에서 파이프라인 변경:
   - DetailModal의 파이프라인 select 편집 시 같은 로직 적용 (Undo 토스트는 칸반에서만).
   - 단계 변경 시 해당 단계의 첫 파이프라인으로 자동 설정.
