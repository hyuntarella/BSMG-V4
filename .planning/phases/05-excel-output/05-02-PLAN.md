<task type="auto">
  <name>Task 2: 동적 행 삽입 + 남는 행 숨김</name>
  <files>lib/excel/generateWorkbook.ts</files>
  <action>
fillTemplateSheet 함수에 동적 행 처리를 추가한다.

1. **템플릿 기본 행 수 감지**: 템플릿 Sheet2에서 아이템 영역(행7~)의 기본 행 수를 감지한다. 방법: 행7부터 아래로 내려가며 B열(sort_order)이 숫자인 행을 카운트하거나, 합계 영역("소계" 등) 시작 행을 찾아서 `templateItemCount = summaryStartRow - 7`.

2. **행 삽입 (아이템 > 템플릿 행 수)**:
   - `const overflow = items.length - templateItemCount`
   - overflow > 0이면, 합계 영역 바로 위에 행을 삽입한다: `ws.spliceRows(summaryStartRow, 0, ...새행들)`.
   - ExcelJS의 `spliceRows`는 행을 삽입할 때 기존 행을 아래로 밀어낸다.
   - 삽입된 행의 서식을 바로 위 행(마지막 아이템 행)에서 복사한다: 각 셀의 font, border, alignment, numFmt를 복사.
   - 합계 영역의 행 번호도 overflow만큼 내려간다.

3. **남는 행 숨김 (아이템 < 템플릿 행 수)**:
   - `const unused = templateItemCount - items.length`
   - unused > 0이면, 사용하지 않는 행(7 + items.length ~ 7 + templateItemCount - 1)을 숨긴다: `ws.getRow(r).hidden = true`.
   - 숨긴 행의 값을 빈 문자열로 클리어한다.

4. **합계 영역 위치 동적 계산**: 합계 행 위치 = 7 + items.length (삽입/숨김 후 기준). 값 채우기는 plan-01에서 만든 로직 재사용.

주의: ExcelJS spliceRows는 셀 병합을 깨뜨릴 수 있다. 템플릿의 합계 영역에 셀 병합이 있으면 병합 정보도 업데이트해야 한다. 합계 영역이 행 이동 시 병합이 깨지면, 병합을 해제 후 다시 설정한다.
