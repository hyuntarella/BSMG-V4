  <action>
generateWorkbook 함수를 전면 리팩토링한다.

1. **템플릿 로드**: `new ExcelJS.Workbook()` 후 `wb.xlsx.readFile(path.join(process.cwd(), 'public/templates/complex-template.xlsx'))`로 템플릿을 로드한다. `import path from 'path'` 추가.

2. **Sheet2(상세) 아이템 채우기**:
   - 템플릿의 Sheet2(두번째 시트, 인덱스 2 또는 이름으로 접근)를 가져온다: `wb.getWorksheet(2)` 또는 이름 기반.
   - 아이템 시작 행은 **행 7**이다 (행 1~6은 템플릿 헤더).
   - 복합 시트(`estimate.sheets`에서 `type === '복합'`인 시트)의 items를 순회하며 각 행에 채운다.
   - 컬럼 매핑 (1-indexed):
     - B(2): sort_order (번호)
     - C(3): name (품명)
     - D(4): spec (규격)
     - E(5): unit (단위)
     - F(6): qty (수량)
     - G(7): mat (재료 단가)
     - H(8): mat_amount (재료비 금액)
     - I(9): labor (노무 단가)
     - J(10): labor_amount (노무비 금액)
     - K(11): exp (경비 단가)
     - L(12): exp_amount (경비 금액)
     - M(13): total (합계)
   - 숫자 값은 number로 넣는다 (문자열 아님). 0인 값은 빈 문자열 `''`로 넣어서 셀이 비어보이게 한다.
   - 기존 템플릿 행의 서식(폰트, 테두리, 정렬)은 건드리지 않는다. ExcelJS는 readFile로 로드하면 기존 서식이 유지된다. 값만 `.value = ...`로 설정한다.

3. **합계 영역 채우기**:
   - 아이템 마지막 행 다음에 합계 영역을 채운다.
   - 합계 시작 행 = 7 + items.length (템플릿에 미리 합계 행이 있으면 그 위치를 사용).
   - `calc(sheet.items)`를 호출하여 CalcResult를 얻는다.
   - 합계 행 레이아웃:
     - 소계 행: C열에 "소계", M열에 calcResult.subtotal
     - 공과잡비 행: C열에 "공과잡비 (3%)", M열에 calcResult.overhead
     - 기업이윤 행: C열에 "기업이윤 (6%)", M열에 calcResult.profit
     - 계 행: C열에 "계", M열에 calcResult.totalBeforeRound
     - 합계 행: C열에 "합계 (10만원 절사)", M열에 calcResult.grandTotal
   - 단, 템플릿에 이미 합계 영역 라벨이 있을 수 있으므로, 먼저 템플릿을 열어서 구조를 확인해야 한다. 실행 시 `wb.getWorksheet(2)`의 행들을 콘솔로 출력하여 구조를 파악한 후 정확한 행 번호를 결정한다.

4. **기존 addCoverSheet, addWorkSheet 함수는 유지**하되 generateWorkbook 내에서 호출하지 않는다 (fallback용으로 남겨둠). 새 함수명은 `generateFromTemplate`로 만들고, `generateWorkbook`에서 템플릿이 있으면 `generateFromTemplate`을, 없으면 기존 로직을 사용하도록 분기한다.

5. **export 시그니처 유지**: `generateWorkbook(estimate: Estimate): Promise<ExcelJS.Workbook>`과 `workbookToBuffer` 시그니처는 그대로 유지하여 generate route.ts 수정 불필요.

주의: `path` 모듈은 Node.js 서버사이드에서만 사용 가능. 이 함수는 API route에서 호출되므로 문제없다.
