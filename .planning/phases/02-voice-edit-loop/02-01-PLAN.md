handleVoiceCommands 내부에서 `voice.playTts`를 호출하는데, 이것은 useVoice의 반환값이다. useVoice와 handleVoiceCommands가 순환 참조가 되지 않도록 주의:
- handleVoiceCommands는 useCallback으로 감싸고, voice.playTts를 직접 참조하지 않고 `playTtsRef`를 사용 (callbacksRef 패턴).
- 또는 더 간단하게: handleVoiceCommands에서 TTS가 필요한 시스템 명령은 addLog만 하고, TTS 재생은 useVoice의 handleModifyResponse/handleCommandResponse에서 이미 처리되므로 중복 TTS 호출을 제거.
- **실제 검토**: EstimateEditor의 handleVoiceCommands에서 voice.playTts를 호출하는 곳은 set_margin, load 등 시스템 명령뿐. 이들은 useVoice의 onCommands 콜백으로 호출되므로, voice 객체가 이미 생성된 후 실행된다. 순환 참조가 아니라 useCallback 의존성 배열에 voice가 있어야 하는 구조.
- **해결**: useEstimateVoice 내부에서 voice를 먼저 생성하고, handleVoiceCommands를 voice 이후에 선언. useVoice의 onCommands에는 ref를 통해 최신 handleVoiceCommands를 전달.

```typescript
useWakeWord({
  onToggle: voice.toggleRecording,
  onWakeWord: () => {
    if (estimate.sheets.length === 0) {
      voiceFlow.startFlow()
    } else {
      voice.startRecording()
    }
  },
  onEditMode: () => {
    if (estimate.sheets.length > 0 && !editMode.isEditMode) {
      editMode.enterEditMode()
    }
  },
  enabled: voice.status === 'idle' || voice.status === 'speaking',
})
```

```typescript
onSttText: (text) => {
  // 수정 모드에서 "그만"/"종료" 감지 -> 수정 모드 해제
  if (editMode.isEditMode && /그만|종료|멈춰/.test(text.trim())) {
    editMode.exitEditMode()
    voice.stopRecording()
    addLog('assistant', '수정 모드 종료.')
    voice.playTts('수정 모드를 종료합니다.')
    return  // LLM으로 보내지 않음
  }
  // 기존 voiceFlow 분기
  if (voiceFlowRef.current.isActive) {
    voiceFlowRef.current.processText(text)
  } else {
    addLog('user', text)
  }
}
```
