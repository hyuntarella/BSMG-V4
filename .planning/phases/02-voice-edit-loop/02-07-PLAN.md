현재 상태:
- useVoiceEditMode.ts에 VAD 코드가 구현되어 있음 (AnalyserNode 기반, 103~171줄).
- 조건: isEditMode && enableVad && voiceStatus === 'recording'

잠재 문제:
1. 듀얼 스트림: 녹음용 MediaRecorder + VAD용 getUserMedia = 마이크 2개 동시 접근.
   갤럭시탭 Chrome은 하나의 getUserMedia만 허용할 수 있음.
2. requestAnimationFrame 루프: 탭 비활성 시 멈춤 → setInterval로 변경 검토.
3. exitEditMode 호출 후 상태 정리: VAD 스트림 해제 확인.

해결 방향:
- 녹음용 MediaRecorder의 stream을 재사용하여 VAD 분석. 별도 getUserMedia 호출 제거.
- MediaRecorder 생성 시 stream을 ref로 저장하고, VAD에서 해당 stream의 AudioContext를 사용.
- 불가능할 경우: enableVad=false를 기본값으로 변경하고, 갤럭시탭 테스트 후 결정.

수정 범위: hooks/useVoiceEditMode.ts만 수정. 단, VAD가 녹음 stream을 재사용하려면
useVoice.ts에서 stream ref를 노출해야 할 수 있음 — 이 경우 useVoice.ts도 최소 수정.
