# 방수명가 견적서 v4 — Claude Code 규칙서

> **스택**: Next.js 14 (App Router) + Supabase + Vercel + OpenAI (STT/TTS) + Claude (LLM)
> **핵심**: 음성 퍼스트 견적서. 터치 0회가 목표.
> **상세 기획**: `docs/PROJECT_SPEC.md` 참조

---

## 1. 절대 규칙

- GAS 코드 수정 금지. v4는 GAS와 완전 독립
- v2 코드 재사용 금지. Figma 디자인만 참조
- 음성이 먼저. 모든 입력/수정은 음성으로 가능해야 한다
- TypeScript strict. any 금지
- 컴포넌트 파일당 200줄 이내
- RLS 필수. 모든 테이블에 company_id 기반 격리
- API 키 하드코딩 금지
- Next.js 15, React 19, Tailwind 4 업그레이드 금지
- 인라인 스타일, CSS 모듈, styled-components 금지. Tailwind만
- 클래스 컴포넌트 금지. 함수형 + Hooks만

---

## 2. 코드 작성 규칙

- 서버 컴포넌트 기본. 'use client'는 클라이언트 상호작용 필요 시에만
- Supabase 클라이언트는 lib/supabase/client.ts, lib/supabase/server.ts 사용. 새 인스턴스 생성 금지
- API 라우트는 app/api/ 하위 route.ts
- 기존 코드의 패턴/네이밍을 따른다
- 기존 코드를 임의로 리팩토링하지 않는다. 지시 범위만 수정
- 새 패키지 설치 전 반드시 보고 + 승인
- 금액 계산 로직 수정 시 기존 계산식 먼저 명시 후 변경 보고
- 절대경로만: @/lib/, @/components/, @/hooks/. 상대경로 금지

---

## 3. 검증 프로토콜

> "괜찮습니다"는 금지. 모든 검증은 구체적 증거 기반이다.

### 코드 검증 (매 작업 완료 시 필수)
1. npm run build 통과
2. npm run lint 에러 0
3. TypeScript 타입 에러 0
4. 테스트 전부 통과해야 다음 단계

### UI 디자인 검증 (UI 작업 시 필수)
1. UI 코드 작성 전, Figma MCP로 대상 컴포넌트의 디자인 토큰을 추출하라 (색상, 폰트, 간격, 정렬, 크기 전부)
2. 코드 작성 후, 생성한 HTML/CSS의 모든 스타일 값을 Figma 토큰과 1:1 대조하라
3. 차이가 있으면 수정 → 다시 대조 → 차이 0개일 때만 완료
4. 대조 결과를 표로 보고하라:
   | 요소 | Figma 값 | 코드 값 | 일치 |
5. 한 컴포넌트씩만 작업하라. 여러 컴포넌트를 한 번에 하지 않는다

## 4. 서브에이전트 규칙

### 지시 시 필수 포함
1. 대상 파일 경로 (full path)
2. 구체적 행동 (무엇을 어떻게)
3. 성공 기준
4. 참조할 기존 코드 패턴
5. 금지 사항

### 완료 시 필수 출력
- 수정한 파일 목록
- 각 파일 변경 요약 (1줄)
- 빌드/린트 통과 여부
- 미해결 이슈

---

## 5. 금지 사항

- 지시 범위 밖 파일 수정 금지
- 테스트 통과시키려고 테스트 자체 수정 금지
- 에러 발생 시 임의 우회 금지. 원인 보고
- 미완료 시 "완료했습니다" 금지. 명시적 보고
- 자가 평가 신뢰 금지. 빌드/린트/테스트/스크린샷 비교만 완료 기준
- "이전에 완료됨" 자기 판단으로 태스크 스킵 금지
