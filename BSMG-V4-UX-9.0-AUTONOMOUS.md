## 사용자 리서치 결과 (모든 단계에 반영할 것)

### 대시보드
- 영업사원 전용. 가장 먼저: 오늘 일정, 견적서 열람한 고객, 연락할 곳, CS DB
- 클릭하면 CRM 카드 상세가 바로 떠야 함 (페이지 이동 아님, 모달/사이드패널)
- 사장/직원 구분 필요 없음

### CRM
- 칸반 드래그 매우 자주 사용 — 필수 편의성
- 카드에 보여야 할 것: 주소(가장 먼저), 금액, 담당자, 견적일자, 견적서 보낸 일자, 노트/댓글
- 검색 기준: 주소
- "다음에 뭘 해야 하지?" 파악: 메모 및 댓글
- 직원별 담당 고객 나눠서 봄
- 고객 카드에서 견적서/제안서로 바로 이동 필수
- 고객 카드에서 Google Drive 링크로 히스토리 확인

### 견적서
- 핵심 = 단가 조정 (작업의 80%)
- 평단가(헤베당 단가), 고객 제공 평단가가 직관적으로 보여야 함
- 숫자/금액 가독성: monospace, 천단위 구분, 우측 정렬
- 수정 요청 30% — 전부 가격 관련
- 현장에서는 못 씀 (PC 전용)
- 완성 후: PDF 다운 + 이메일 전송

### 캘린더
- 일정 등록이 구글캘린더보다 쉬워야 함
- CRM 정보 자동 입력 (중복 입력 제거)
- 캘린더만 보고 영업 다닐 수 있어야 함
- 일간=주간>월간 순서로 자주 봄
- 직원별로 따로 봄
- 고객 정보/견적서/Google Drive로 바로 이동 가능해야 함
- 필수 정보: 주소, 시간, 메모/댓글

### 모바일
- 견적서/제안서 작성 제외 전부 모바일에서 되어야 함
- 일정 확인, 고객 검색이 주 사용

### 알림
- 잔금 입금, 견적서 열람이 최우선
- 앱 내 + 카톡
- 직원에게: 연락할 곳, 견적서 열람한 고객

### 최대 불만
- 매번 이동하고 로딩되는 순간 → 속도가 핵심

## 1단계: 비주얼 디자인 업그레이드

### 목표
shadcn/ui 디자인 토큰을 Tailwind config에 반영. 컴포넌트 직접 설치 아님.

### 작업

**1-1. 디자인 토큰**
- globals.css 또는 tailwind.config에 CSS 변수 추가
- --background, --foreground, --card, --card-foreground, --primary, --primary-foreground, --secondary, --muted, --muted-foreground, --border, --ring, --destructive, --success
- 다크/라이트 모드 지원 (현재 다크 헤더가 있으므로 기존 brand 토큰과 충돌 없이 통합)

**1-2. 공통 컴포넌트 스타일**
- 버튼: primary/secondary/outline/ghost/destructive 변형
- 카드: 일관된 라운딩(rounded-lg), 그림자(shadow-sm), 보더(border)
- 모달/사이드패널: backdrop blur, 애니메이션
- 인풋: focus ring, placeholder 스타일
- 뱃지: 파이프라인 단계별 색상 코딩
- 테이블: 줄 간 구분, hover 효과
- 토스트: 성공(green)/에러(red)/경고(yellow)/정보(blue)

**1-3. 대시보드**
- 카드 위젯 스타일 통일 (오늘 일정, 견적서 열람, 연락할 곳, CS DB)
- 숫자 강조 (text-2xl font-bold)
- 카드 클릭 시 CRM 상세가 모달/사이드패널로 열리는 구조 준비 (스타일만)

**1-4. CRM 칸반**
- 카드: 주소 굵게(font-semibold), 금액/담당자/날짜 서브텍스트(text-sm text-muted-foreground)
- 파이프라인 컬럼 헤더: 건수 뱃지 + 색상 코딩
- 드래그 중 카드 그림자 강화
- 직원별 필터 드롭다운 스타일

**1-5. 캘린더**
- 타임그리드 가독성: 시간 라벨, 현재 시간선 강조
- 이벤트 카드: 타입별 색상 (방문=blue, 시공=green, 미팅=purple, 기타=gray)
- 일간/주간 뷰 우선 최적화 (가장 자주 사용)

**1-6. 견적서**
- 금액: font-mono, 천단위 구분, 우측 정렬
- 평단가/헤베당 단가 표시 영역 시각적 강조 (배경색 구분)
- 공종 행 hover/선택 상태 명확화

**1-7. 로딩 스켈레톤**
- 모든 데이터 fetch 페이지에 스켈레톤 추가
- CRM: 칸반 카드 스켈레톤
- 캘린더: 타임그리드 스켈레톤
- 대시보드: 카드 위젯 스켈레톤
- 견적서: 테이블 행 스켈레톤

**1-8. 모바일 반응형**
- 칸반: 가로 스크롤 (snap scroll)
- 캘린더: 터치 스와이프로 날짜 이동
- 네비게이션: 모바일에서 하단 탭바 (대시보드/CRM/캘린더/견적서/더보기)
- 카드/모달: 모바일에서 full-width

### 완료 시

```bash
# 빌드 + 테스트 확인
npm run build
npx vitest run
npx playwright test --project=chromium

# 커밋
git add -A
git commit -m "style: shadcn/ui 디자인 토큰 적용 + 전체 비주얼 업그레이드"
git push origin main
```

## 2단계: UX 기능 품질 (10라운드 반복)

### 목표
기계적으로 측정 가능한 UX 품질을 올린다. 각 라운드에서 3~5개 개선.

### 라운드 구조 (10회 반복)

**🎯 기획자**: 아래 체크리스트에서 미달 항목 선택 → 개선안 정의
**💻 개발자**: 구현
**🔍 검수자**: 코드 리뷰 + 기존 테스트 확인
**🤖 테스터**: 빌드 + vitest + E2E 실행
**📊 평가자**: 10점 만점 채점 → 9.0 이상이면 조기 종료

### UX 체크리스트 (Yes/No로 판단 가능한 것만)

**로딩/성능**
- [ ] 모든 페이지에 로딩 스켈레톤이 있는가
- [ ] 데이터 fetch 중 사용자에게 피드백이 있는가
- [ ] 페이지 이동 시 300ms 이내에 첫 콘텐츠가 보이는가
- [ ] 무한 스크롤 또는 페이지네이션이 필요한 목록에 적용되었는가
- [ ] 이미지 lazy loading이 적용되었는가

**에러/빈 상태**
- [ ] 모든 API 에러 시 사용자에게 토스트/메시지가 표시되는가
- [ ] 데이터가 없을 때 빈 상태 UI가 있는가 (예: "아직 고객이 없습니다")
- [ ] 네트워크 오프라인 시 안내가 있는가
- [ ] 폼 입력 유효성 에러가 인라인으로 표시되는가

**인터랙션**
- [ ] 모든 클릭 가능한 요소에 hover/active 상태가 있는가
- [ ] 모달/패널 닫기가 ESC + 바깥 클릭으로 되는가
- [ ] 삭제/위험 동작에 확인 다이얼로그가 있는가
- [ ] 성공 동작에 피드백이 있는가 (저장됨, 삭제됨 등)
- [ ] 드래그앤드롭에 시각적 피드백이 있는가 (CRM 칸반)

**접근성**
- [ ] 모든 인풋에 label이 있는가
- [ ] 탭 키로 주요 동작을 수행할 수 있는가
- [ ] 색상만으로 정보를 구분하지 않는가 (아이콘/텍스트 병행)
- [ ] 폰트 크기가 최소 14px인가

**모바일**
- [ ] 모든 터치 타겟이 44px 이상인가
- [ ] 가로 스크롤이 의도치 않게 발생하지 않는가 (칸반 제외)
- [ ] 모바일에서 하단 탭바 네비게이션이 있는가
- [ ] 모달이 모바일에서 전체 화면으로 열리는가
- [ ] 핸드폰에서 고객 검색이 편리한가

**정보 밀도**
- [ ] 대시보드에서 오늘 할 일을 한 화면에서 파악할 수 있는가
- [ ] CRM 카드에 핵심 정보(주소/금액/담당자/날짜)가 모두 보이는가
- [ ] 견적서에서 평단가/헤베당 단가가 직관적으로 보이는가
- [ ] 캘린더 이벤트에 주소/시간/메모가 보이는가

**네비게이션/연결**
- [ ] 대시보드 → CRM 카드 상세를 클릭 한 번에 볼 수 있는가
- [ ] CRM 카드 → 견적서/제안서로 바로 이동할 수 있는가
- [ ] CRM 카드 → Google Drive 링크가 있는가
- [ ] 캘린더 이벤트 → CRM 카드로 이동할 수 있는가
- [ ] 캘린더 이벤트 → Google Drive 링크가 있는가
- [ ] 뒤로가기가 자연스러운가

## 3단계: 정보설계 반영 (사용자 리서치 기반)

2단계에서 커버 안 된 사용자 리서치 항목을 마무리한다.

### 3-1. 대시보드 정보 재배치
- 최상단: 오늘 일정 (시간순)
- 그 아래: 견적서 열람한 고객 목록 (열람 시간순)
- 그 아래: 연락해야 할 고객 (CS DB, 먼저연락X 단계)
- 각 항목 클릭 → CRM 카드 상세 사이드패널 (페이지 이동 아님)

### 3-2. CRM 카드 정보 우선순위
- 1순위: 주소 (font-semibold, 첫 줄)
- 2순위: 견적금액 (font-mono, 우측)
- 3순위: 담당자 뱃지
- 4순위: 견적일자 / 견적서 보낸 일자
- 5순위: 최근 메모/댓글 미리보기 (1줄)
- 카드 내 아이콘 링크: 견적서, 제안서, Google Drive

### 3-3. 견적서 단가 표시
- 상단 또는 사이드바에 항시 표시: 총 합계, 평단가(합계÷면적), 헤베당 단가
- 공종 변경/단가 수정 시 실시간 업데이트
- 배경색으로 구분 (muted 배경)

### 3-4. 캘린더 일정 정보
- 이벤트 카드에 표시: 주소, 시간, 고객명
- 이벤트 클릭 → 상세 패널에 CRM 링크 + Google Drive 링크
- 일정 등록 시 CRM 고객 선택하면 주소/전화번호 자동 입력

### 3-5. 모바일 하단 탭바
- 5개 탭: 대시보드, CRM, 캘린더, 견적서, 더보기(제안서/설정)
- 현재 페이지 탭 하이라이트
- PC에서는 숨김 (기존 사이드바 유지)

### 3단계 완료 시

```bash
npm run build
npx vitest run
npx playwright test --project=chromium

git add -A
git commit -m "feat: UX 정보설계 반영 — 대시보드/CRM/견적서/캘린더 사용자 리서치 적용"
git push origin main
```
