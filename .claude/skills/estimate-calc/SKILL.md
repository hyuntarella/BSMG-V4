# 견적 계산 스킬

Estimate Engine 에이전트가 사용하는 작업 스킬.

## 계산 흐름

```
1. P매트릭스 조회 → 면적범위로 행 선택
2. 공종별: mat × qty + labor × qty + exp × qty = 소계
3. 전체 소계 합산 (장비류 제외분 별도 집계)
4. 공과잡비 = (소계 - 장비류) × 3%
5. 기업이윤 = (소계 - 장비류) × 6%
6. 총액 = 소계 + 공과잡비 + 이윤
7. 최종금액 = Math.floor(총액 / 100000) × 100000  (10만원 절사)
```

## 계산 로직 수정 프로토콜

CLAUDE.md 절대 규칙: **금액 계산 로직 수정 시 기존 계산식 먼저 명시 후 변경 보고**

수정 시 반드시 다음 형식으로 보고:

```markdown
### 계산식 변경 보고
| 항목 | 기존 | 변경 후 | 사유 |
|------|------|--------|------|
| 공과잡비 | (소계-장비) × 3% | (소계-장비) × 3.5% | 규정 변경 |
```

## 핵심 파일별 역할

| 파일 | 역할 | 수정 시 주의 |
|------|------|------------|
| `calc.ts` | 소계→공과잡비→이윤→절사 | 절사 단위 변경은 전체 금액에 영향 |
| `buildItems.ts` | 공종 CRUD, P매트릭스 매핑 | 면적범위 경계값 테스트 필수 |
| `priceData.ts` | P매트릭스 데이터 로딩 | 캐시 무효화 고려 |
| `costBreakdown.ts` | 비용 분류 | 장비류 판별 로직 확인 |
| `types.ts` | Estimate/Sheet/Item 타입 | DB 스키마와 동기화 |

## 면적범위 매핑

P매트릭스는 면적 구간별로 다른 단가를 제공한다. 경계값에서의 동작:
- 정확히 경계값일 때 → 상위 구간 적용 (예: 100m²이면 100~200 구간)
- 매핑 실패 시 → 에러 보고, 기본값 적용 금지

## 금지 사항
- 계산식 변경 후 기존 계산식 미명시 금지
- 절사 로직 제거 금지
- 공과잡비/이윤 비율 무단 변경 금지 (사용자 승인 필요)
- 음수 금액 자동 보정 금지 (원인 보고)
